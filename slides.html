Sampling

Stratified sampling

  • Problem 1 — load all 40M rows: 40M rows × 24 columns ≈ 6–8 GB with pandas. Slow and memory-intensive.
  • Problem 2 — read first 200k rows: data is sorted chronologically. First 200k = Oct 21 midnight only. Biased sample, no later dates or hours.
  • Solution — stratified sampling: load all 40M rows, group by day, sample exactly 20,000 rows per day. Within each day, preserve the 16.98% click ratio — same proportion of clicks and non-clicks as the full dataset.
  • Result: exactly 20,000 rows per day, Oct 21–30. CTR locked at 16.98%. Fully reproducible every run.
EDA

Key findings

  • CTR = 16.98% — class imbalance. 83% of rows are non-clicks. Accuracy is a misleading metric.
  • device_id = a99f214a — appears in 165k of 200k rows. Anonymous user placeholder, not a real device.
  • C20 contains -1 — Avazu's null marker, not a real integer.
  • Hour CTR: 15.5%–19.1% — modest signal from time of day.
  • App traffic CTR: ~20% vs site traffic: ~13%.
  • 14 of 24 columns have 100+ unique values — high cardinality, requires frequency encoding.

Feature engineering

24 raw columns → 52 features

  • Frequency encoding (14 columns) — replace each category ID with log(count + 1) from training set. site_id has ~1,800 unique values; frequency encoding keeps it as 1 column.
  • One-hot encoding (7 columns) — for low-cardinality columns like device_type (4 values). Drop first category to avoid multicollinearity. 7 columns → 36 binary features.
  • Z-score scaling — mean 0, std 1. Fit on train only, applied to val and test. Prevents leakage.
  • Leakage rule: all encoding maps and scaling parameters computed on training rows only.

2 time features (hour of day, day of week) + 14 freq + 36 one-hot = 52 features total.

Models

Four models

  • Naive Bayes (baseline) — Bernoulli, Laplace smoothing, log-space. Binarizes all features at 0. Log-loss 0.549, AUC 0.615.
  • Logistic Regression — mini-batch SGD (batch=2048), L1/L2 regularization, learning rate decay. Grid search: 18 combinations. Log-loss 0.436, AUC 0.647.
  • Decision Tree — Gini impurity, max_depth=6, min_samples_leaf=50, 20 sampled thresholds per split. Log-loss 0.430, AUC 0.666.
  • Random Forest — 10 trees, bootstrap sampling, √52 ≈ 7 features per split. Log-loss 0.432, AUC 0.677.

Threshold tuning

Default threshold of 0.5 fails at 17% CTR

  • Model outputs cluster around 0.15–0.25. Nothing crosses 0.5.
  • At threshold 0.5, Logistic Regression: recall = 0.74%. Model almost never predicts click=1.
  • Fix: sweep thresholds 0.05–0.60 on val set, pick the one maximising F1.
