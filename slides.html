Sampling

Reservoir sampling

  • Problem 1 — load all 40M rows: 40M × 59 features × 4 bytes ≈ 8 GB. Exceeds available RAM.
  • Problem 2 — read first 200k rows: data is sorted chronologically. First 200k = Oct 21 midnight only. Biased sample, no later dates or hours.
  • Solution — reservoir sampling: stream all 40M rows one at a time. Maintain a 200k reservoir. Each new row randomly replaces an existing entry. Every row has equal probability 200k / 40M of being selected.
  • Result: ~20k rows per day, Oct 21–30. Uniform across all hours and dates.

Feature engineering

24 raw columns → 59 features

  • Frequency encoding (14 columns) — replace each category ID with log(count + 1) from training set. site_id has ~1,800 unique values; one-hot would create 1,800 columns. Frequency encoding: 1 column.
  • One-hot encoding (7 columns) — for low-cardinality columns like device_type (4 values). Avoids implying false numeric ordering.
  • Z-score scaling — mean 0, std 1. Fit on train only, applied to val and test. Speeds up gradient descent.
  • Leakage rule: all encoding maps and scaling parameters computed on training rows only.
Models

Four models

  • Naive Bayes (baseline) — Bernoulli, Laplace smoothing, log-space. Binarizes all features at 0. Log-loss 0.539, AUC 0.625.
  • Logistic Regression — mini-batch SGD (batch=2048), L1/L2 regularization, learning rate decay. Grid search: 18 combinations. Log-loss 0.411, AUC 0.661.
  • Decision Tree — Gini impurity, max_depth=6, min_samples_leaf=50, 20 sampled thresholds per split. Log-loss 0.407, AUC 0.671.
  • Random Forest — 10 trees, bootstrap sampling, √59 ≈ 7 features per split. Log-loss 0.411, AUC 0.677, PR-AUC 0.288.

Threshold tuning

Default threshold of 0.5 fails at 17% CTR

  • Model outputs cluster around 0.15–0.25. Nothing crosses 0.5.
  • At threshold 0.5: recall = 0.3%. Model never predicts click=1.
  • Fix: sweep thresholds 0.05–0.60 on val set, pick the one maximising F1.
